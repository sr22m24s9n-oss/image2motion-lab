有力仮説 = **設定の二層構造**:
- 「エージェントの設定」のデフォルトは**エージェント経由の生成**に効く
- 編集画面下の**プロンプトボックス/編集ボックスは都度選択**で、初期値がOmni Flash。
  ここから生成が走ると、エージェント側のデフォルト（9:16・Veo Fast）は素通りされる
- 今回のショット1は編集ボックス表示がOmni Flashだった → この入り口で走った疑い
- 判定方法: 消費クレジット差分（15cr=Omni Flash / 20cr=Veo 3.1 Fast）

なお「混雑等でモデルが勝手に切り替わる」挙動は調査でも報告を見ていない。可能性は低い。

**判定（同日）: 残高935→915=−20cr → モデルはVeo 3.1 Fastで走っていた。**
「モデル巻き戻り」説は棄却。落ちたのは縦横比だけ。
Ingredients(材料から動画)は9:16ネイティブ対応済みと確認（Google公式ブログ・
2026年時点の解説記事）ため、モード制約説も薄い。
残る有力候補 = **プロンプトボックス自体の縦横比設定（都度選択・16:9初期値）**。
エージェントの設定パネルの9:16は、この入り口には効いていなかった可能性。
→ 再生成時は「生成に使う入り口の設定アイコン」で9:16を直接指定して検証。

## 公式ヘルプの発見: 延長の実行モデルは選べない（Veo 3.1 Lite 固定らしい）

モデル×機能対応表を確認:
- **動画の拡張/延長: Veo 3.1 Lite のみ対応（8秒）**。Fast/Qualityは「提供なし」、
  Omni Flashは「近日予定」
- 拡張できるのは「Veo 3.1系で生成した動画のみ」→ テスト04クリップ（Veo Fast産）は対象内
- 編集画面下のボックス（Omni Flash表示）は**「動画→動画編集」用**。
  これはOmni Flashの専有機能で、延長とは別の道具
- ヘルプには「選択モデルで使えない機能は互換モデルに自動切替」の記述もある（検索要約より）

つまり「延長でVeo Fastを選べない」のは仕様。延長パートは**Lite品質で出る**可能性が高い。
本体Fast・継ぎ足しLiteの品質差が目立つかは観測項目に追加。Liteの消費crも未知（残高差分で実測）。

**実地確認（08:24）**: 延長UIのチップは「続きを生成（Veo 3.1 - Lite）」表示 = 公式表どおり。

## 警告「1個の参照が無効になっています」（08:24）

延長プロンプトに引き継がれた**キャラ資産（材料参照）が、延長モードでは使えず無効化された**もの。
延長はテキスト＋前クリップの最終1秒だけで生成し、材料参照を受け付けない（表の
「動画の素材/参照」と「拡張」が別機能であることと整合）。
- 影響: **固有性の保持が「最終1秒の引き継ぎ」だけに乗る**。材料の後ろ盾なしで
  オッドアイ・装飾・画風が維持されるかが新しい観測項目
- ブロッカーではない見込み（生成自体は可能）→ このまま実行して観測する

## 延長も実行されず（09:25 判定）

延長を送信後、かなり待っても**続きが生成されない・クレジット消費もなし**（残高915のまま）。
タイムラインはシーン合計16秒表示になり、8秒以降が灰色の空き枠のまま
（延長のプレースホルダだけ置かれてジョブが走っていない状態に見える）。

**本日二度目の「宣言と実行の乖離」。** 今回の有力容疑者は**無効化された参照**:
「1個の参照が無効になっています」を残したまま送信した結果、
リクエストが裏で弾かれた可能性（先の「ブロッカーではない見込み」は外れの疑い）。

### 領主の切り分け（09:30）: 空プロンプトでも「1個の参照が無効」が出る

新しいシーンにクリップを置き、**プロンプトを一切入れない状態**でも警告が出ることを確認。
= 無効参照はプロンプト欄の添付ではなく、**元クリップに紐づく材料参照**
（クリップの生成履歴が持つキャラ資産）を指している。外す×も存在しない。

**昇格した仮説: 材料(Ingredients)で生成したクリップは、延長と非互換**
（参照を引き継げず無効化 → ジョブが黙殺される）。
公式表の「動画の素材/参照」と「拡張」が別機能である事実とも整合。

**戦略的な含意**: 本プロジェクトのクリップは固有性固定のため**全て材料付き**で作る。
仮説が正なら延長機能はこのパイプラインと恒久的に非互換 → **連結の主路はプランB
（フレーム渡し）に確定**する。検証実験（材料なしLiteクリップ10cr+延長10crで
仮説の白黒をつける）は方法論の完全性のためのオプション。

### 切り分け手順（安い順）
1. ページをリロード → 灰色枠が埋まっていないか確認
2. 延長の残骸（灰色枠）を削除 → ＋→拡張をやり直し。今度は**無効参照を×で外してから**送信
3. それでも動かなければ、歌詞・引用符なしの一文プロンプトで再試行（プロンプト内容の切り分け）
4. 全滅なら延長は当面見限り、**プランB=フレーム渡し**へ:
   クリップ最終フレームを取り出し「フレームから動画」の開始フレームに指定して
   Veo 3.1 Fastで新規生成（20cr）。延長と同じ連続性を機能側で確保する
