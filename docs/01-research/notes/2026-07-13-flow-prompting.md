# 2026-07-13 Flow の操作方法と Veo 3.1 プロンプトの書き方（調査）

一次情報は Google Cloud 公式ブログ「Veo 3.1 のプロンプトに関する究極のガイド」。
それを軸に、2026年前半の日本語解説記事で補完した。

## 1. Flow 画面の基本操作（画像から動画を作る手順）

1. プロジェクト画面下部の**プロンプトボックス**で生成モードを選ぶ。
   画像起点なら「**フレームから動画**」を選択
2. 「開始フレーム」ボックスをクリックし、PCから画像をアップロード
   （終了フレームも指定可能。アップロード後に切り抜き画面が出るので範囲を選んで保存）
3. テキスト欄には「**画像から何が起こるか**」（動き）を書いて生成
4. モデル・画質（Fast / Quality）や縦横比は プロンプトボックスの**設定（スライダー型アイコン）**から選ぶ
5. 生成後は**シーンビルダー**に追加してクリップの延長・連結ができる

補足: 右側のアシスタントチャット（スクショで開いていたパネル）には
「コンセプトをプロンプトに変換する」「ストーリーボードを作成する」があり、
**下手に手で書くよりまずこれに下書きさせる**のが公式導線として用意されている。

## 2. プロンプトの公式（Google 公式ガイド）

基本形:

```
[映像(カメラワーク・ショット構成)] + [被写体] + [アクション] + [コンテキスト(背景)] + [スタイルと雰囲気(照明・ムード)]
```

- 目安は**英語で50〜150語**（英語推奨。短すぎは情報不足、長すぎは指示が埋もれる）
- 「美しい」「素晴らしい」など主観形容詞は効かない。**具体的な描写**で書く
- 迷ったら1行ずつ: 被写体 → 動き → カメラ → 固定したい要素 → 光や音

### 画像から動画（今回の本命ケース）の鉄則

> **画像がすでにあるなら、画像の内容を再説明しない。**
> プロンプトが制御するのは「動き・カメラ・雰囲気の変化・音」だけ。

被写体・画風は入力画像が決めるので、テキストで二重に書くと崩れの原因になる。

### 音声の書式（Veo 3.1 はネイティブ音声対応）

| 種類 | 書き方 |
| --- | --- |
| 会話 | 引用符で台詞を書く（例: 少女が「行こう」と言う） |
| 効果音 | `SFX: 遠くで雷が鳴る` |
| 環境音 | `周囲の音: 静かな風のうなり` |

### ネガティブプロンプト

「〜なし」という漠然とした除外ではなく、**望む状態を具体的に描く**。
❌「人工構造物なし」→ ✅「建物や道路のない荒涼とした風景」

### タイムスタンプ・プロンプト

`[00:00-00:02]` 形式で時間を区切り、1回の生成（8秒）内にマルチショット構成を作れる。
1分ショートのカット割りをクリップ内でも刻めるということ。

## 3. 1分ショートへの応用方針（仮説）

- 8秒クリップ × 8本前後。**「最初と最後のフレーム」機能**で前クリップの最終フレームを
  次クリップの開始フレームに渡すと、つなぎ目の一貫性が取りやすい
- キャラの一貫性には「**材料から動画（Ingredients）**」でキャラ参照画像を渡す手もある
  （アニメ系イラストでどこまで効くかは実地テスト項目）
- 複雑な内容を1プロンプトに詰めず、**複数ステップに分割**するのが公式の推奨姿勢

## 4. テスト時に検証すること

- [ ] アシスタントの「コンセプトをプロンプトに変換する」の出力品質（日本語入力→英語プロンプト化まで任せられるか）
- [ ] i2vで「動きだけ書く」鉄則がアニメ系イラストでも成立するか
- [ ] タイムスタンプ指定の効き具合
- [ ] Fast と Quality の品質差とクレジット実消費
