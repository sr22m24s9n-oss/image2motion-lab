# 2026-07-13 Flow キャラクター機能 — 「キャラクター情報」欄の作用先（実地+調査）

テスト素材（キャラ1枚絵）をキャラクター資産として登録した際の画面から確認。

## キャラクター資産は三点で構成される

| 項目 | 作用先 |
| --- | --- |
| 登録画像 + ボディを作成 | **見た目**。シーン間でキャラの外見を一貫させる参照。「ボディを作成」は全身参照を補完し一貫性を上げる |
| 音声を選択 | **声**。台詞を生成させたときの音声 |
| キャラクター情報（省略可） | **演技**。画面注記のとおり「Flow エージェントはこの情報を使用して、キャラクターが登場するシーンを作成します」— つまり絵柄ではなく、エージェントがシーンを組むときの台詞・仕草・振る舞いの条件付け |

要するに、キャラクター情報欄は**画像生成プロンプトではなく、キャラの人格シート（システムプロンプト相当）**。
空でも動きテストは可能。エージェント任せでシーンを作らせる段階で効いてくる。

## 運用方針

- 言語分担: キャラクター情報欄は日本語で書いてよい（エージェントは Gemini 基盤で日本語対応）。
  生成プロンプト本体は英語推奨のままなので、**日本語の意図 → 英語プロンプト変換は AI（本リポジトリの対話側）が担う**
- 書く内容は「性格 / 口調 / 動きの癖 / 決め仕草」程度の短いシートで足りる
- モデル表記メモ: キャラ画像の編集は Nano Banana 2（画像モデル）、動画生成は Veo 系と役割が分かれている
