# 方法論ドラフト01 — クリップ連結の設計原則（2026-07-14）

テスト01〜04の結果と対談から。連結テスト（テスト05）で検証後に確定版へ。

## 原則: 繋がりは「指示」ではなく「機能」で作る

各生成は前のクリップの記憶を持たない。プロンプトに「さっきの続きで」と書いても、
参照（フレームや材料）が渡っていなければ効かない。

| 繋げたい状況 | 使う機能 | プロンプトに書くこと |
| --- | --- | --- |
| 同じシーンの続き（動作の連続） | シーンビルダーの**延長**（前クリップの最終フレームを引き継ぐ） | 「次の8秒で何が起こるか」だけ |
| シーンをまたぐ（場所・時間が変わる） | **ジャンプカット**＝独立生成で割り切る | 新しいショットとして通常どおり |
| どのクリップでも | キャラ資産を**材料添付**（固有性の固定） | 見た目の再説明はしない |

映像はもともとカット割りの文化なので、全クリップを滑らかに繋ぐ必要はない。
「シーン内は延長、シーン間はカット」が基本形。

## 制作手順（1分ショート）

1. **ショット表（絵コンテ）を先に作る** — 8秒×7〜8本に分解し、各ショットの
   「場所・動き・カメラ」を1行ずつ。シーン内連続か、カット切替かを印つけ
2. ショットごとに生成（キャラ材料添付＋日本語指示、モデルはVeo 3.1 Fast=20cr/本）
3. シーン内連続の箇所だけ延長機能で繋ぐ
4. シーンビルダーで並べ、書き出し
5. **音は編集段で一本化** — クリップ個別の歌に頼らず、通しの歌/BGMを
   編集ツール（Canva/CapCut等）で1本重ねる。クリップ側の音は環境音程度に使うか捨てる。
   口の動きと歌詞の完全一致は狙わない（アニメ調なら許容されやすい）

## 費用の目安

8本構成 + リテイク2〜3本 ≒ 200〜220クレジット/作品（月1,000で4作品前後）
