# PDF報告書の設計 — ミッション完了の最終成果物（2026-07-15）

領主裁定: **PDF形式の報告書をもってミッションコンプリート**。
求められる水準: 完璧ではなく「**狙った操作ができる程度のコツ**」まで目途を立てて
説明できること。特に延長部の破綻（人物が透けて消える等）への対処法則を含む。

## 書名（案）

「画像1枚から1分ショートを作る — Google Flow (Veo 3.1 Lite) 実践ガイド」

## 構成（全10章 + 付録）

1. **表紙・この資料について** — 対象読者、2026-07-15時点の情報である旨
2. **できること・できないこと** — 期待値の線引きを最初に正直に
   （細部は揺れる／物理の辻褄は完璧にならない／ガチャ性はある。その上で狙いは通せる）
3. **最初の準備（1回だけ）** — アカウント、無料枠/Pro、キャラクター登録、初期設定
   （出力数1x・9:16・エージェントへの手順）
4. **基本の一本を作る** — 材料から動画＋プロンプトの型（動き・カメラ・音だけ書く）
5. **続きを伸ばす（延長）** — ★本書の心臓部。延長プロンプトの法則集
   （本日の追加実験の結果をここに反映）
6. **場面を切り替える（ジャンプカット）** — シーン内=延長／シーン間=カットの使い分け
7. **60秒に組み立てる** — ショット表の書き方、費用の目安
8. **書き出しと仕上げ** — ダウンロード、音の一本化（編集アプリ）
9. **公開のルール** — AI生成ラベル、商用利用、SynthID
10. **困ったときの早見表** — 黙殺・縦横比・安全フィルタ・英語化の対処
- **付録A**: コピペ用プロンプト文例集
- **付録B**: なぜLiteなのか（Fastとの使い分け・連結不可の報告）

## 「コツ」の記述スタイル（5章の型）

各法則を以下の4点セットで書く（実験の再現性を読者に渡す）:

> **法則N: （短い命令形）**
> - なぜ: （理由を1行）
> - ❌ 悪い例: （プロンプト原文）→ こうなる
> - ✅ 良い例: （プロンプト原文）→ こうなる

## 延長実験の報告テンプレート（領主→AI の受け渡し用）

実験1回につきこの4点をもらえれば、そのまま法則化して5章へ組み込める:

```
目的: （何を試したか。例: 人物を維持したまま猫を出す）
プロンプト: （原文）
結果: （何が起きたか。維持できたもの/崩れたもの）
消費: （cr）
```

## 検証候補（延長プロンプトの仮説。安い順に）

1. **主語を人物に固定し続ける** — 「彼女は歩き続ける。その足元を黒い猫が横切る」
   （新要素を主語にせず、人物の継続動作を必ず書く。省略すると人物が"用済み"扱いされて
   消される仮説）
2. **1延長1イベント** — 足す出来事は1つだけ。2つ以上詰めると破綻率が上がる仮説
3. **位置関係を明示** — 「画面手前を」「彼女の背後の壁沿いを」など、新要素の通り道を指定
4. **カメラを止める** — 延長部ではカメラ指示を「そのまま」にする（カメラ移動と
   新要素の同時要求が崩れの元という仮説）

## 生成の段取り

1. 領主の延長実験 → 結果報告（上のテンプレート）
2. AIが法則集を確定 → 教材（02-lite-route-teaching-guide.md）へ反映
3. 全章をPDF化（AI側でレイアウト・目次付きで出力）→ 領主の検収 → 完了
