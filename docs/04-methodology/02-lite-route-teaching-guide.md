# 1分ショートの作り方 — Lite路線・教材版（2026-07-15）

知人に教えるための手順書。**2026-07-15 領主裁定: 教材のデフォルトはLite路線**。
終了条件: ①この手順でショートが開通していること ②「Fastだと連結が上手くいかない」の
報告が整っていること（→ 末尾の「Fastについて」）。

## 前提

- Google Flow（labs.google/flow）にGoogleアカウントでログイン
- **Google AI Pro契約なら月1,000クレジット**。Lite生成は10cr/本
- 未契約でも**無料枠（1日50crの報告あり・Lite使用可）**で回せる可能性
  → Lite 10cr/本なら1日5本。1分ショート（8本+リテイク）は2〜3日で無料完走の計算
  （※無料枠の正確な条件は知人のアカウントで要実地確認）
- 用意するもの: 動かしたい画像1枚（**権利がクリアなもの**）

## 手順

### 1. キャラクターを登録する（最初に1回だけ）

1. Flowの「キャラクター」から知人の画像をアップロードし、キャラクター資産を作る
2. 音声を選ぶ（**注意: 声の年齢の印象が見た目に波及した疑い例あり。落ち着いた声を推奨**）
3. 性格などの情報欄は空でもよい（絵柄には影響しない）

### 2. プロジェクトの初期設定（最初に1回だけ）

1. **エージェントの設定**: 生成前の確認=常時／動画・画像の出力数=**1x**（事故防止）／
   縦横比=**9:16**（YouTubeショート用）
2. **エージェントへの手順**（常設指示欄）に登録:
   - 動画は常に1本だけ生成する
   - 縦横比は常に9:16
   - 動画モデルは常に **Veo 3.1 Lite**
   - 人物クリップではキャラクター資産を必ず材料として使う。見た目の再説明はしない
   - 画風は元イラストの調子のまま（例: 手描き厚塗り、水彩、セル画調…実物に合わせる）

### 3. 最初のクリップを作る（10cr）

- 生成モードは**材料から動画**でキャラクター資産を添付
- プロンプトは**動き・カメラ・音だけ**書く（見た目は画像が決める。二重に書くと崩れる）
- 型: [カメラ] + [動き] + [場所] + [音] + [画風固定の一文]。日本語でOK
- 例:
  「石畳の路地。彼女はゆっくり歩き出し、くるりと回って軽やかに踊る。
  カメラはゆっくり周回。音: 靴音が石壁に反響、遠くの街のざわめき。
  画風は元イラストのまま。」

### 4. 続きを伸ばす（延長・各10cr）

1. クリップをシーンビルダーに追加 → クリップ末尾の**「＋」→「拡張」**
2. プロンプトには**次の7〜8秒で起こることだけ**書く（前の説明を繰り返さない）
3. 「1個の参照が無効になっています」と出るが**無視してよい**（延長には材料を
   引き継げないという告知。生成は走る）

**柵（重要）**: 延長部分では**人物を大きく映さない**（引き・後ろ姿・風景や小物のカットに）。
材料の後ろ盾がないため、顔の細部が変わったり、物が透けるなどの崩れが出やすい。

### 5. 場面を変える（ジャンプカット）

場所や時間が変わるカットは、延長ではなく**新規生成**（手順3と同じ・材料添付）で作り、
シーンビルダーで並べる。映像はもともとカット割りの文化なので、全部を滑らかに
繋ぐ必要はない。**シーン内=延長／シーン間=ジャンプカット**が基本形。

### 6. 60秒の組み立て

1. 先に**ショット表**を書く: 8秒×8本に分解し、各行「場所・動き・カメラ」1行ずつ。
   延長で継ぐか、カットで切り替えるかを印つけ
2. 表のとおりに生成 → シーンビルダーで並べる
3. 費用目安: Lite主体で 8本×10cr + リテイク ≒ **100〜130cr/作品**

### 7. 書き出しと仕上げ

1. ⬇ボタンでダウンロード（解像度・透かしの有無はここで確認）
2. **音は編集アプリで一本化**: クリップごとの音はガイド扱いにして、通しのBGM/歌を
   CapCut・Canva等で1本重ねる（8秒ごとに音が変わる問題の回避）
3. 60秒に整えて完成

### 8. 公開するとき

- YouTubeへは**「改変コンテンツ（AI生成）」ラベルを付けて**アップロード（2026年の方針）
- Veo出力の商用利用・収益化は基本OK。SynthID（不可視透かし）は全出力に入っており
  消せない（消そうとするのは規約違反）

## トラブル対応の柵

| 症状 | 対処 |
| --- | --- |
| 「生成しています」のまま動かない | **クレジット残高を見る**。減っていなければ走っていない（黙殺）。文言を変えて再依頼 or「新規に1本」と明示 |
| 縦横比が無視される | 生成に使う**その入り口**の設定アイコンで9:16を直接指定（設定は入り口ごとに別） |
| 危険・恐怖系の描写で拒否される | 「追う・怯える・逃げる」を避け、「静かに・穏やかに」の語彙に書き換える |
| 歌が英語になる | 歌詞そのものを引用符で渡す（「彼女は日本語でこう歌う——『…』」）※検証途上 |
| 顔や飾りが元絵と違う | 材料添付を確認。細部（左右で違う目の色など）は**Liteでは落ちることがある**と先に伝える |

## Fastについて（終了条件②の報告）

- Veo 3.1 Fast（20cr/本）は単発の画質・細部忠実度でLiteに勝る（オッドアイ等の
  細部まで固定できた実績あり）
- しかし**Fastで作ったクリップは延長（拡張）に対応していない**（公式のモデル×機能表で
  「Fast: 提供なし」。実測でもジョブが黙殺され、エラーすら出ない）
- よって**連結して長尺を作る用途ではLite一択**。Fastは「1本もの8秒を最高画質で」の
  用途に限って使う
